# 출근 마일리지 (Commute Mileage) — Spec

## 개요

매일 아침 환경(미세먼지·강수·풍속·UV·기상특보) 데이터를 출근 난이도 점수로 환산하고, 그 점수에 어울리는 영웅 칭호와 1줄 서사를 AI로 생성하는 RPG형 미니 웹앱이다. 사용자는 한국 광역시 중 한 곳을 골라 오늘의 점수 카드를 받고, 매일의 점수가 캘린더에 누적된다.

## 범위

### 포함
- 메인 페이지: 도시 선택 → 오늘의 점수·칭호·서사 카드
- 마일리지 페이지: 일자별 칭호 캘린더 + 누적 점수 합계
- 점수 산출(미세먼지·강수·풍속·UV·기상특보 가중 합) — 출근 시간대 07:00~10:00 평균 기준
- AI 생성 칭호·1줄 서사 — Gemini Flash, 같은 날 1회만 호출하고 결과 보존
- 일자별 점수·칭호 누적 — 디바이스 로컬 (단일 사용자·단일 디바이스 가정)
- 공유 — 카드 텍스트(점수·칭호·서사·날짜·도시)를 클립보드로 복사
- 한국 광역시 8곳(서울/부산/대구/인천/광주/대전/울산/세종) 드롭다운

### 제외
- **친구 랭킹·소셜·로그인** — 4시간 내 인증·DB 동기화 비용 과다
- **일러스트/이미지 자동 생성** — 외부 이미지 API 의존, 실패 모드 다수
- **푸시 알림·이메일** — 매일 방문 모델로 충분
- **동네/주소 자동 검색** — 외부 키 발급 비용. 도시 드롭다운으로 시연 가치 동일
- **다국어** — 한국어 only
- **다중 디바이스 동기화** — 디바이스 1개의 누적 가치만 시연

## 시나리오

### 1. 첫 방문 — 도시 선택 후 점수 카드 발급
- **Given** — 누적 기록이 없는 사용자가 메인 페이지에 진입한 상태. "도시를 선택하세요" 안내가 보이고 도시 드롭다운에 8개 옵션이 있다.
- **When** — 사용자가 드롭다운에서 "서울"을 선택한다.
- **Then** — 카드 영역에 (오늘 날짜·도시명 "서울"·환경 난이도 점수 한 정수·영웅 칭호 한 문구·1줄 서사 한 문장) 다섯 요소가 한 카드 안에 표시된다.

성공 기준:
- [ ] 도시 "서울" 선택 → 카드에 오늘 날짜(YYYY년 M월 D일 형식)·"서울" 텍스트가 보인다
- [ ] 카드에 0 이상 정수 점수가 표시된다 (예: "오늘 난이도 47")
- [ ] 카드에 칭호 텍스트가 한 문구로 표시된다 (예: "안개 속의 검객")
- [ ] 카드에 1줄 서사가 한 문장으로 표시된다 (마침표 포함)
- [ ] 점수 카드 전체가 모바일 viewport(360×640px) 한 화면에 잘림 없이 들어간다

### 2. 같은 날 재방문 — 점수·칭호 보존
- **Given** — "서울"에서 오늘 카드를 이미 한 번 받은 사용자가 페이지를 새로고침한다.
- **When** — 페이지가 다시 로드된다.
- **Then** — 동일한 점수·칭호·서사 카드가 즉시 표시된다 (다시 AI를 호출해 다른 칭호가 나오지 않는다).

성공 기준:
- [ ] 새로고침 전 카드의 점수와 새로고침 후 카드의 점수가 동일하다
- [ ] 새로고침 전 칭호 텍스트와 새로고침 후 칭호 텍스트가 동일하다
- [ ] 새로고침 전 서사 텍스트와 새로고침 후 서사 텍스트가 동일하다

### 3. 다음날 첫 방문 — 새 점수 발급 + 누적 +1
- **Given** — 어제 "서울"에서 카드를 받은 사용자가 자정을 지나 다음날 페이지에 진입한다.
- **When** — 페이지가 로드되고 사용자가 같은 도시("서울")를 유지한다.
- **Then** — 어제와 다른 날짜의 새 카드가 생성된다. 마일리지 페이지의 일자 수가 어제 대비 +1, 점수 합계가 오늘 점수만큼 증가한다.

성공 기준:
- [ ] 새 카드의 날짜가 어제 카드의 날짜보다 하루 뒤다
- [ ] 마일리지 페이지의 "총 출근일" 수가 어제보다 1 증가한다
- [ ] 마일리지 페이지의 "누적 점수" 합계가 오늘 카드의 점수만큼 증가한다
- [ ] 캘린더에서 어제 일자에는 어제 칭호, 오늘 일자에는 오늘 칭호가 표시된다

### 4. 마일리지 페이지 — 캘린더 + 누적 점수
- **Given** — 최소 2일 이상의 점수 기록이 누적된 사용자가 마일리지 페이지로 이동한다.
- **When** — 마일리지 페이지가 로드된다.
- **Then** — 캘린더에서 점수가 기록된 날짜 셀에 칭호(또는 칭호의 짧은 표시)가 보이고, 페이지 상단에 (총 출근일·누적 점수 합계) 두 통계가 표시된다.

성공 기준:
- [ ] 점수 기록이 있는 일자의 캘린더 셀에 칭호 텍스트(또는 단축 표시)가 보인다
- [ ] 점수 기록이 없는 일자의 캘린더 셀에는 칭호 표시가 없다
- [ ] "총 출근일" 통계가 기록된 일자 수와 일치한다
- [ ] "누적 점수" 통계가 기록된 일자들의 점수 합계와 일치한다

### 5. 데이터 fetch 실패 — 명시적 에러 표시
- **Given** — 외부 환경 데이터 호출이 응답을 주지 못하는 상태(타임아웃 또는 오류 응답).
- **When** — 사용자가 도시를 선택한다.
- **Then** — 카드 자리에 "환경 데이터를 받지 못했어요. 잠시 후 다시 시도해주세요." 메시지와 재시도 버튼이 표시된다. 점수·칭호·서사는 생성되지 않고 누적에도 기록되지 않는다.

성공 기준:
- [ ] 환경 데이터 fetch 실패 시 카드 자리에 위 에러 메시지가 보인다
- [ ] 같은 상황에서 "다시 시도" 버튼이 표시된다
- [ ] 같은 상황에서 마일리지 페이지의 총 출근일·누적 점수가 변하지 않는다

### 6. AI 생성 실패 — 기본 칭호로 대체
- **Given** — 환경 데이터는 정상 수신했지만 칭호·서사 생성 호출이 실패한 상태.
- **When** — 카드가 표시될 시점이 된다.
- **Then** — 점수는 정상 계산되어 카드에 표시되고, 칭호 자리에는 점수 구간별 기본 칭호("오늘의 출근자" / "끈기 있는 출근자" / "역경의 출근자" 등), 서사 자리에는 점수 기반 기본 한 줄이 표시된다. 사용자는 에러 화면을 보지 않는다.

성공 기준:
- [ ] AI 호출 실패 시 카드에 점수는 정상 표시된다
- [ ] AI 호출 실패 시 카드에 점수 구간별 기본 칭호가 칭호 자리에 표시된다
- [ ] AI 호출 실패 시 카드에 점수 기반 기본 서사 한 문장이 서사 자리에 표시된다
- [ ] AI 호출 실패 시 카드 어디에도 "에러" "실패" 문구가 표시되지 않는다

### 7. 공유 — 카드 텍스트 클립보드 복사
- **Given** — 오늘 카드가 표시된 상태(점수·칭호·서사·날짜·도시 모두 채워짐).
- **When** — 사용자가 카드의 "공유" 버튼을 클릭한다.
- **Then** — 카드 다섯 요소(날짜·도시·점수·칭호·서사)가 줄바꿈으로 정돈된 텍스트가 클립보드에 복사되고, 사용자에게 "복사되었어요" 짧은 피드백이 보인다.

성공 기준:
- [ ] "공유" 버튼 클릭 후 클립보드 텍스트에 오늘 날짜·도시명·점수·칭호·서사가 모두 포함된다
- [ ] 클릭 후 "복사되었어요" 또는 동등한 확인 피드백이 1~2초간 표시된다

## 불변 규칙

### 데이터 일관성
- 같은 날(자정 ~ 자정)에는 같은 도시 선택 시 점수·칭호·서사 세 값이 항상 동일하다 (재호출하지 않는다).
- 마일리지 페이지의 "총 출근일"·"누적 점수"는 캘린더에 표시된 점수 기록과 항상 일치한다.

### 프라이버시
- 사용자 식별 정보(이름·이메일·정확한 위치)를 입력받지 않는다. 도시 단위 선택만 받는다.
- 누적 기록은 디바이스 로컬에만 보관한다. 외부 서버로 전송하지 않는다.

### 성능 / UX
- 도시 선택 후 카드의 점수·날짜·도시는 환경 데이터 수신 후 즉시 표시된다(스켈레톤 1초 이내).
- 점수 카드 전체가 모바일 viewport(360×640px) 한 화면에 잘림 없이 들어간다.

## 의존성
- 외부 환경 데이터 소스(시간별 미세먼지·기상·UV)
- 외부 AI 텍스트 생성 서비스(점수·환경 컨텍스트 → 칭호·1줄 서사)
- 디바이스 로컬 영구 저장소(브라우저 세션 간 일자별 점수·칭호 보존)

## 미결정 항목
- **도시 변경의 같은 날 처리 방향** — 사용자가 점수 받은 뒤 같은 날 다른 도시로 바꾸면 (a) 점수 보존(현 가정) (b) 새 도시로 재계산 + 캘린더에 도시 표기 변경 (c) 추가 카드를 새로 발급. 1차는 (a)로 진행, sketch 단계에서 한 번 더 검토.
- **점수 가중치 공식의 정수 변환** — 가중합을 그대로 표시할지(예: 47.3 → 47), 0~100으로 normalize할지. 가중치 표는 plan 단계에서 확정.
